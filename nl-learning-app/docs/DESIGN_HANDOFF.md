# 🎨 Дизайн-документация приложения «Учим нидерландский»

> **Для дизайнера / дизайн-ИИ.** Этот документ описывает ВСЁ приложение так, чтобы можно
> было полностью переделать визуал и стиль, **не трогая обучающий материал, порядок уроков
> и логику обучения**. Читай раздел «⛔ Красные линии» до того, как что-то менять.

Дата: 2026-07-11 · Версия контента: CHANGELOG (26) в приложении.

---

## 1. Что это за приложение

- **Продукт:** бесплатное мобильное веб-приложение для изучения нидерландского языка.
- **Аудитория:** русскоязычные, живущие в Нидерландах. Цель пользователя — сдать экзамен
  **инбургеринга** (inburgering) и выучить язык до уровня A1 → A2 → B1.
- **Весь интерфейс на русском**, обучающий контент — нидерландский. **Украинского языка в
  приложении нет и быть не должно.**
- **Монетизация (будущая):** PRO-подписка €9.99/мес, 14 дней бесплатно. Главная PRO-фишка —
  AI-репетитор.
- **Технически:** один самодостаточный файл `nl-learning-app/app.html` (~5.7 МБ). Ванильный
  HTML + CSS + JS, без фреймворков и без внешних запросов (шрифты вшиты base64). Открывается
  как обычная веб-страница в браузере телефона. Публикуется как artifact.

### Как посмотреть приложение вживую
- **Живая ссылка (артефакт):** https://claude.ai/code/artifact/5a496423-0dce-422e-9814-15f41092dd90
- **Локально:** просто открыть `nl-learning-app/app.html` в браузере. Лучший вид — в
  мобильном режиме (DevTools → эмуляция телефона, ширина ≤ 520px: приложение разворачивается
  на весь экран без рамки-«телефона»).
- На десктопе (> 520px) приложение показано внутри макета телефона (класс `.phone`,
  392×806px) — это just для превью, на самом телефоне рамки нет.

## 2. ⛔ Красные линии — что НЕЛЬЗЯ трогать

Дизайн меняется **только в CSS и в разметке-обёртках**. Ниже — то, что определяет обучение;
это должно остаться нетронутым:

| Нельзя менять | Где это | Почему |
|---|---|---|
| **Тексты уроков, слова, переводы, вопросы** | всё внутри `const DATA=`, `var BANK=`, `var WL=`, `var WLX=`, `var THEMES=`, `var KNM=`, `var CROSSWORDS=`, `META`, и все файлы в `data/*.json` | Это обучающий материал. Опечатка ломает учёбу. |
| **Порядок и структура уроков** | массив `THEMES` (порядок тем A1→A2→B1), поля `lessons:[...]`, номера уроков, `META`, `themeLessons` | Порядок изучения продуман. Перестановка = сломанная методика. |
| **Логика движка заданий** | функции `renderStep`, `bindStep`, `checkOpt`, `finish`, `next`, `markWrong`, `startPart` | Определяет, как проверяются ответы, «сердечки», повтор ошибок. |
| **Прогресс и разблокировки (гейтинг)** | `a1Passed()`, `b1Passed()`, `lessonState`, `saveState`/`loadState`, `progress.*`, экзамены | Определяет, что и когда открывается. |
| **Сборка** | `build.py`, `tools/*.py`, JSON в `data/` | app.html **генерируется** скриптом. Правки логики нужно делать так, чтобы пересборка их не стёрла (см. §3). |
| **Встроенные шрифты (base64-блоки)** | огромные строки `@font-face … base64` в `<head>` | Это данные шрифтов Mont и Evolventa. НЕ удалять. Можно менять только *где* они применяются (`font-family`), см. §5. |

**Короткое правило:** трогаешь `<style>` и HTML-классы — можно. Трогаешь JS-данные, номера
уроков, тексты, логику — нельзя.

## 3. Архитектура файлов (чтобы правки не потерялись)

`app.html` **собирается автоматически** из данных скриптом `build.py`. Он заменяет
размеченные регионы файла содержимым из `data/*.json`. Если поправить эти регионы вручную,
следующая пересборка их перезапишет.

```
data/*.json  ──(tools/make_data.py, gen_*.py)──▶  bank.json, lessons_generated.json, …
                                                          │
                                                   build.py собирает
                                                          ▼
                                                     app.html
```

### Что из этого важно дизайнеру
- **Дизайн (`<style>`, шрифты-применение, разметка экранов в JS-функциях) НЕ перезаписывается
  сборкой** — он лежит в статической части `app.html` (примерно строки 23–900: стили, иконки,
  функции экранов). Это ваша зона.
- **Данные (тексты уроков, слова)** — перезаписываются сборкой. Не ваша зона.
- Граница в `build.py` — «регионы» между якорями `H_MAN / H_TT / H_BASE / H_GEN / H_DLG /
  H_WL` и якорь `function renderNode(`. Всё, что относится к дизайну, находится **после**
  `renderNode` (стили — вообще в `<head>`), поэтому пересборка их не трогает.

> **Практический совет для дизайн-ИИ:** правь только блок `<style>…</style>` в `<head>` и,
> при необходимости, HTML-строки внутри функций экранов (`home`, `profile`, `dict` и т.д.).
> После правок можно НЕ запускать `build.py` — дизайн от него не зависит. Если сборку всё же
> запустят, стили и вёрстка экранов сохранятся.

## 4. Дизайн-система (design tokens)

Всё завязано на **CSS-переменные** в `:root` (строки 24–36 в `app.html`). **Меняя их, вы
меняете весь визуал разом** — это самый безопасный способ рестайла. Есть светлая и тёмная
темы: значения переменных переопределяются в `@media (prefers-color-scheme:dark)` и в
`:root[data-theme="dark"]` / `["light"]` (ручной переключатель темы).

### Цвета (светлая тема → тёмная)
| Токен | Назначение | Light | Dark |
|---|---|---|---|
| `--oranje` / `--oranje-2` | **бренд, акцент**, «пройдено», стрик, кнопки-действия | `#F26A16` / `#D8560A` | `#FF894A` / `#F26A16` |
| `--delft` / `--delft-2` | **основной синий**, текущий урок, ссылки, primary | `#2C6BD6` / `#1E509F` | `#5D97E8` / `#3B78D4` |
| `--berry` | фиолетовый акцент (SRS-повторение, лиги) | `#7C4DE0` | `#A78BFA` |
| `--bg-a` / `--bg-b` | фон страницы (градиент) | `#F1F3F7` / `#E4E8F0` | `#0A0D13` / `#11151E` |
| `--surface` / `--surface-2` | стекло карточек (полупрозрачные) | белый .76 / .52 | тёмный .68 / .42 |
| `--brd` | границы | `rgba(18,24,38,.10)` | `rgba(255,255,255,.10)` |
| `--ink` / `--ink-2` / `--ink-3` | текст: основной / вторичный / третичный | `#0F1726` / `#586074` / `#8A93A6` | `#EDF1F8` / `#A2ABBE` / `#6E7789` |
| `--success` (+`-s`) | «верно», прогресс | `#12A150` | `#3ECB7E` |
| `--danger` (+`-s`) | «неверно», сердечки | `#E23B2E` | `#F0705C` |
| `--tint` | лёгкая подсветка фона | оранжевая .06 | оранжевая .06 |

- Цветовые токены с суффиксом `-s` — это полупрозрачная «мягкая» заливка того же цвета
  (для фонов бейджей/плашек).
- Многие компоненты используют `color-mix(in srgb, var(--delft) 10%, …)` — динамическое
  смешение. Если меняете палитру через переменные, миксы подстроятся сами.

### Тени и типографика
| Токен | Значение (light) | Назначение |
|---|---|---|
| `--sh-sm` | мягкая маленькая | карточки, кнопки |
| `--sh-md` | средняя | «поднятые» карточки, банеры |
| `--sh-lg` | большая | рамка телефона, оверлеи |
| `--font` | `'Evolventa', -apple-system, …` | базовый шрифт всего UI |

### Радиусы (не токенизированы, но система есть)
Крупные карточки `20px`, кнопки/инпуты `14–15px`, узлы карты `22px` (квадратные) или `50%`
(круглые), плашки-«таблетки» `999px`. Единый скругляющий язык — сохраняйте его.

### Шрифты (важно — сохранить обязательно)
- **Evolventa** — основной шрифт всего интерфейса (кириллица), `--font`.
- **Mont** — для латиницы и цифр в заголовках/акцентах. Применяется точечно (строка 199):
  ```css
  h1,h2,h3,.qt,.btn,.kick,.bdg,.appbar b,.brand b,.wnl,.dnl,.speakphrase,
  .dsec>summary,.done-wrap h2,.stats .s,.lv,.tcard b,.partcard b { font-family:'Mont','Evolventa',var(--font) }
  ```
  Идея: русский текст — Evolventa, нидерландские слова и числа в заголовках — Mont (латиница
  у Mont красивее). **Оба шрифта вшиты base64 и должны остаться.** Менять можно только список
  селекторов, где применяется Mont, если захотите иную типографику. Кегли/жирность — свободно.

## 5. Что можно менять свободно (зона дизайнера)

1. **Все значения CSS-переменных** в `:root` и в тёмной теме (§4) — палитра, тени.
2. **Любые правила внутри `<style>`** — отступы, радиусы, размеры, анимации, раскладка.
3. **Классы компонентов** (см. каталог §7) — их оформление.
4. **Анимации** (`@keyframes`: `fadeup`, `pop`, `shake`, `breathe`, `slidein`, `splashin`,
   `cfall` и т.д.) — можно перерисовать или убрать. Учтите `@media (prefers-reduced-motion)`.
5. **HTML-разметку/классы внутри функций экранов** (`home()`, `profile()`, `dict()` …) — можно
   менять структуру блоков и классы, **если не трогать данные и вызовы логики** (`data-*`
   атрибуты обработчиков, вызовы `startPart`, `go`, `bindNav` и пр. должны сохраниться).
6. **Иконки** — объект `I={…}` (строка 263) содержит inline-SVG (play, heart, home, user,
   check, x, lock, mic). Можно заменить на свои SVG.
7. **Splash-экран, конфетти, тосты** — чисто визуальные, меняйте смело.

> ⚠️ Если меняете разметку экрана, **сохраняйте `data-*`-атрибуты** (`data-tab`, `data-go`,
> `data-part`, `data-say`, `data-b1skip`, `data-finala2` и т.п.) и `id` (`cta`, `ansline`,
> `bank`, `aichat`…) — на них висят обработчики. Классы и вложенность — свободно.

## 6. Экраны приложения (карта интерфейса)

Приложение — одно-страничное (SPA). Экраны рисуются JS-функциями в контейнер `#screen`
(класс `.screen` внутри `.phone`). Внизу — таб-бар навигации (`nav()`), сверху на главных
экранах — шапка (`appbar()`).

### Навигация (нижний таб-бар `.nav`, функция `nav()` / `bindNav()`)
5 вкладок: **Учить** (`learn`→`home`) · **Тренир.** (`train`) · **Лига** (`league`) ·
**Лента** (`feed`) · **Профиль** (`me`→`profile`).

## 9. Тёмная тема — обязательна к сохранению

Приложение полностью поддерживает светлую и тёмную темы:
- **Авто** — по системной теме (`@media (prefers-color-scheme:dark)`).
- **Ручной переключатель** в Настройках → ставит `data-theme="dark"|"light"` на `<html>`;
  правила `:root[data-theme="dark"]` / `["light"]` перекрывают системные.
- Логика: `applyTheme()` (строка 753) и обработчик `themeBtn` (655).

**Требование:** любой новый дизайн должен одинаково хорошо выглядеть в обеих темах. Держите
цвета в переменных и дублируйте их для dark — тогда всё работает само. Не хардкодьте
`#fff`/`#000` в компонентах — используйте `--ink`, `--surface`, `--bg-*`.

## 10. Рекомендованный порядок работы для дизайн-ИИ

1. Открой живой артефакт (или `app.html` локально в мобильном режиме) и пройди все экраны
   из §6 — увидишь реальный визуал.
2. Начни с **токенов** (§4): палитра, тени, радиусы, типографика. Смена переменных в `:root`
   (и dark-варианте) уже даёт новый облик без риска для логики.
3. Дальше — **покомпонентно** по каталогу §7: переоформи `.card`, `.btn`, `.dnode`, `.opt`,
   `.foot`, `.appbar`, `.nav`. Проверяй каждую в light и dark.
4. При изменении разметки экранов — сохраняй `data-*`/`id`/вызовы функций (§5).
5. **Не запускай** `build.py` и **не редактируй** `data/*.json`, `DATA/BANK/WL/THEMES/META`,
   номера уроков и функции движка/гейтинга (§2).
6. Тестовый чек-лист перед сдачей: обе темы; мобильный (≤520px) и превью-телефон; экран урока
   со всеми состояниями `.opt` (sel/correct/wrong); экран «готово» с конфетти; карта с замками
   уровней; словарь со сворачиванием.

### Дизайн-инварианты (просьба владельца)
- Бренд-эмодзи/логотип — стропвафель 🧇 (оранжевая гамма). Оранжевый — фирменный акцент.
