# PROJECT CONTEXT — приложение «Учим нидерландский»

> Единый контекст проекта для любого ассистента (Claude Code / ChatGPT / человек).
> Читать первым. Прототип — это **UX/UI-референс**, а не финальная архитектура.

---

## 1. Цель приложения
Бесплатное мобильное приложение для **изучения нидерландского языка на русском**.
Главная ценность: довести пользователя до реального владения языком (говорить, читать,
писать, слушать) и до уровня, достаточного для сдачи **инбургеринга** (A2 → B1).
Модель дохода — подписка **PRO 9,99 €/мес, первые 14 дней бесплатно**.

## 2. Целевая аудитория
Русскоязычные люди в Нидерландах (иммигранты, переехавшие), которым нужно
выучить нидерландский и сдать инбургеринг. Разный стартовый уровень — от нуля до A2.
Тёплый канал привлечения уже есть: сообщество/бот **«Подслушано в Нидерландах»**
(Instagram + Telegram-бот `podslushano-bot`).

## 3. Основные функции
- **Онбординг-воронка:** приветствие → цель → уровень → формат (мин/день) → экран PRO.
- **Тест на уровень** (placement test, 12 вопросов по нарастанию сложности → A1/A2/B1).
- **Карта обучения** в стиле Duolingo: темы → уроки → части; узлы (урок/практика/сундук/трофей).
- **Уроки**, разбитые на **части** (2–5), каждая часть проходится отдельно и насыщена.
- **11 типов заданий:** объяснение, новое слово, выбор, картинка, аудирование,
  впиши-на-слух, перевод, верно/неверно, собери-фразу, соедини-пары, говорение.
- **SRS (интервальные повторения)** — «Умное повторение», сила памяти слова, даты повтора.
- **Словарь A1 по темам** (банк слов) + экран «Память».
- **Геймификация:** XP, стрик 🔥, алмазы 💎, сундуки, лиги, лента, друзья, достижения.
- **AI-репетитор:** ролевые диалоги, объяснение грамматики/ошибок, проверка письма и
  произношения (в прототипе — мок; в проде — реальная LLM).
- **Инбургеринг-трекер:** навыки экзамена Lezen / Luisteren / Spreken / Schrijven / KNM.
- Озвучка (TTS) и распознавание речи (STT).

## 4. Текущий стек (ПРОТОТИП)
- **Один самодостаточный HTML-файл** (`docs/reference/appprototype.html`, ~197 KB):
  инлайновые `<style>` + `<script>` на **чистом ванильном JS**. Без сборки, без фреймворка,
  без внешних зависимостей (CSP артефакта блокирует внешние ресурсы).
- Аудио: браузерный **SpeechSynthesis** (`nl-NL`) для кнопки ▶.
- Распознавание речи: браузерный **SpeechRecognition** для 🎤 (говорение).
- Весь контент (уроки, слова) **захардкожен в JS** внутри файла. Всё состояние
  (прогресс, SRS, XP) — **в памяти**, сбрасывается при перезагрузке. Бэкенда нет.

## 5. Рекомендуемая целевая архитектура (НЕ прототип)
> Это ориентир для настоящей разработки, не обязателен буквально.
- **Клиент:** мобильное приложение — React Native / Expo (или Flutter). Веб-версия — React/Next.js.
- **Контент вынести из кода в данные:** JSON/YAML (уроки, части, задания, словарь по темам).
  Не хранить весь контент в одном HTML/TSX-файле.
- **Бэкенд:** аутентификация, хранение прогресса и SRS, соц-функции (лига/лента/друзья), платежи (Stripe).
- **AI-репетитор:** реальная LLM — **Claude** (та же модель, что в Telegram-боте) через сервер.
- **TTS/STT в проде:** нормальные сервисы (проф. озвучка + распознавание речи типа Whisper).
- **SRS:** реальные даты/персистентность (алгоритм типа SM-2 / Лейтнер).

## 6. Как устроен прототип внутри (карта кода)
Один файл, логика в `<script>`. Ключевые структуры данных:
- `META` — массив из 20 уроков `{n, t, blk}` (blk = 1 или 2, «раздел»).
- `DATA` — объект `{ номерУрока: { parts: [ { t:'название части', steps:[ …задания… ] } ] } }`.
- `THEMES` — 8 тем-модулей `{name, emoji, c(цвет), lessons:[номера]}` (тема 1 = уроки 1–10,
  тема 2 = 11–20, темы 3–8 пока пустые = «скоро»).
- `TRAIN` — наборы тренировок. `TEST` — вопросы теста уровня.
- `SRS` — карта памяти слов `{ nl: {tr, box(1–6), due(день), wrong} }`; `vDay` — виртуальный день.
- `THEMES_A1` — банк слов по 8 темам (сейчас 160 слов, из TaalCompleet A1).
- `progress = {doneParts:[]}` (ключи вида `"урок.часть"`), `skillStat`, `XP`, `STREAK`, `gems`.

**Типы шагов (steps) урока:** `explain`, `word`, `quiz`, `pick`, `listen`, `gapType`,
`translate`, `tf`, `build`, `match`, `speak`.

**Экраны — это функции**, которые рендерят HTML в `#screen` через `innerHTML`
(без фреймворка). Роутер — функция `go(name)`. Главные функции:
`splash`, `onboard`, `testStart/renderTest/testResult`, `home` (карта),
`partsScreen`, `startPart/renderStep/bindStep/donePart` (движок урока),
`review` (SRS), `memory`, `dict`, `train`, `league`, `feed`, `friends`,
`profile`, `aiScreen`, `chestCard/srsCard/renderNode`.

## 7. Данные и источники контента
- **TaalCompleet A1 — тематический словарь** (официальный список слов для инбургеринга):
  8 тем (Hallo!, De school, Wonen, Eten en drinken, De dokter, De kleding, Reizen, Vrije tijd),
  ~600 слов, под-юниты вида `1.1, 1.2 …`. Загружен пользователем (txt + json, без переводов —
  переводы генерируются ИИ). **Это каркас структуры курса A1.**
- **Грамматика:** книга С. А. Миронова «Нидерландский (голландский) язык. Грамматический
  очерк» — источник грамматических правил (переписываются своими словами, не копируются дословно).
- Цель по объёму: **≥ 1500 слов в A1**, сотни предложений/заданий на слово (через AI-конвейер).

## 8. Главный UX/UI-референс
- **`docs/reference/appprototype.html`** — текущий рабочий прототип (открывается в любом браузере,
  на телефоне разворачивается на весь экран). Использовать как ориентир по экранам и потокам.
- **Duolingo** — вдохновение по структуре (карта, юниты, стрик, лиги, сундуки).
  ⚠️ ВАЖНО: **нельзя делать клон Duolingo** — нужен собственный визуальный стиль/бренд
  (владелец отметил, что текущая карта выглядит как плагиат).

## 9. Экраны (есть / планируются)

экран частей урока, урок (11 типов заданий), экран завершения части, SRS-повторение,
«Память», «Словарь A1 по темам», тренировки, лига, лента, друзья, профиль (навыки+достижения),
AI-репетитор (мок), сундуки/алмазы.
Планируется/доработать: тесты юнитов («трофей»), ежедневная цель + гейт по времени,
реальный бэкенд соц-функций и платежей, реальная LLM для AI, собственный визуальный стиль.
